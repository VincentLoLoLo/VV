# 神秘客任務中心

這個專案提供一個靜態入口網站，方便在 n8n 流程中蒐集神秘客任務的兩階段回報：第一次回報一次建立基本資料與營業所服務品質、第二次回報追蹤顧問聯繫結果。頁面預設可部署於單機環境，後續可視需求擴充驗證機制。

## 使用方式

1. 將 `index.html` 發佈至單機主機或任何可提供靜態網站的服務（如 Netlify、Vercel、GitHub Pages）。
2. 在 n8n 建立兩個 webhook 節點：第一次回報（基本資料＋營業所服務品質）、第二次回報（D+3 追蹤）。
3. 分別將 webhook URL 填入 `index.html` 末段的 `initialWebhookUrl` 與 `followupWebhookUrl` 常數。
4. 若需要上傳附件，請在 n8n workflow 中加入合適的檔案儲存邏輯（例如 Google Drive、S3 或資料庫）。
5. 於 workflow 內使用神秘客代號 + 場次日期檢查是否重複提交，避免單機部署時出現重覆回報。
6. 發佈後提供網址給神秘客，即可透過頁面表單提交資料。

## 表單欄位

- 第一次回報：神秘客代號、姓名、職業、聯絡方式、Email、對專案的理解程度、自我注意重點、專業強項、訪客資料、據點與賞車/試乘時間、顧問資訊、接待流程紀錄、提醒與陪同情況、整體體驗評估、推薦意願、驚艷亮點、顧問稱呼、改善建議、各服務環節重要程度與實際體驗評分、佐證檔案。
- 第二次回報：神秘客代號、顧問是否聯繫、聯繫方式、聯繫內容摘要、佐證檔案、補充備註。

## 客製化建議

- 可依任務需求調整頁面文字與欄位名稱。
- 需要驗證身份或登入機制時，可於 n8n workflow 中增加驗證節點，或改用具有認證的前端框架。
- 若需寄送提醒，可在 n8n 排程於 D+3 日發送 email/LINE 通知神秘客再次填寫回報。
- 可於兩個回報的 webhook 中加入資料庫查詢節點，檢查是否已存在相同神秘客代號與場次紀錄，降低重複填寫的困擾。

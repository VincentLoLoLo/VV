# 神秘客預約系統

這個專案提供一個靜態的神秘客入口網站，整合登入、預約時段與新成員註冊。所有流程皆以模擬資料運作，方便先在單機環境驗證介面與自動化流程，再導入 n8n 或其他後端服務。

## 介面特色

- **雙重驗證登入**：神秘客須輸入 Email、密碼並通過 6 位數驗證碼後才能瀏覽預約功能。
- **品牌儀表板首頁**：登入後左側列出 MAZDA、Škoda、Toyota、Volvo、Audi、Honda、Ford、Volkswagen、Hyundai 等在台車廠，顯示每個品牌的任務數量與最近開放的預約時段。
- **專案導覽面板**：選擇品牌後，即時在同頁顯示該品牌的任務目標、統計摘要與專案卡片，作為預約入口指引。
- **專案與預約清單**：每個品牌當季僅保留一台目標車，進入任務卡片後會看到依月份匯整的經銷商卡片（11 月 6 家、12 月 13 家），僅顯示可執行月份與據點名稱，避免洩漏真實日期。
- **PM 專用後台儀表板**：管理員登入後可在「專案總覽」頁面查看各品牌釋出場次、已預約/待預約統計與涵蓋月份，並可一鍵跳轉至對應品牌的儀表板。所有指標目前以模擬數據呈現，日後串接後端即可替換為正式資料。
- **新成員註冊**：提供註冊表單蒐集姓名、聯絡方式、銀行資訊與登入密碼，送出後提示等待審核。
- **即時提示**：操作過程透過 Toast 顯示成功或錯誤訊息，協助測試流程。

## 測試帳號

專案提供模擬帳號以便驗證登入與預約流程，帳號資訊僅供開發測試之用。如需取得最新的測試 Email、密碼與驗證碼，請聯繫專案開發者或系統管理員。

> ⚠️ 實際導入時請改為後端產生一次性驗證碼，並透過 email/SMS 發送給神秘客。

## 部署與客製化

1. 將 `index.html` 上傳至任意靜態網站服務或單機伺服器即可使用。頁面預設載入 Tailwind CDN，需保持可連線狀態。
2. 在 `index.html` 中調整 `BRAND_DATA`、`BRAND_PROJECTS`、`SEASONAL_LOCATIONS` 與 `PROJECT_AVAILABILITY` 等模擬資料，即可快速修改品牌陣列、任務摘要、月度經銷商卡片與可預約時段。
3. 管理員儀表板會依據 `MOCK_SLOTS_DATA` 與 `BOOKING_LOG` 自動產出品牌匯總，導入正式後端時只要將這兩個來源替換為 API 回傳即可沿用現有圖表與表格。
4. 若要串接 n8n 或其他後端，可將登入、預約與註冊處的模擬 `setTimeout` 改為實際的 API 呼叫，並在成功回應後維持目前的 UI 更新流程。
5. 需要正式的雙重驗證時，請於後端產生驗證碼並透過 email/SMS 發送，再將回傳結果與目前的 6 位數欄位連動。

## 與 n8n 整合建議

- 於 n8n workflow 接收預約或註冊表單時，可新增帳號審核、時段鎖定、重複場次檢查等節點，確保單機部署也能避免重複填寫。
- 可使用 n8n 產生一次性驗證碼並寄送給神秘客，再於登入流程中驗證，取代目前的模擬碼。
- 若要導入原先的多階段回報表單，可在登入後導向另一個頁面或嵌入額外的表單卡片，以便重用現有的身分驗證流程。

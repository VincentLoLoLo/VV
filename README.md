# Škoda 神秘客預約系統

這個專案提供一個靜態的神秘客入口網站，整合登入、預約時段與新成員註冊。所有流程皆以模擬資料運作，方便先在單機環境驗證介面與自動化流程，再導入 n8n 或其他後端服務。

## 介面特色

- **雙重驗證登入**：神秘客須輸入 Email、密碼並通過 6 位數驗證碼後才能瀏覽預約時段。
- **預約時段清單**：登入後可檢視示範用的專案場次，點擊即可開啟預約確認視窗。
- **新成員註冊**：提供註冊表單蒐集姓名、聯絡方式、銀行資訊與登入密碼，送出後提示等待審核。
- **即時提示**：操作過程透過 Toast 顯示成功或錯誤訊息，協助測試流程。

## 測試帳號

專案提供模擬帳號以便驗證登入與預約流程，帳號資訊僅供開發測試之用。如需取得最新的測試 Email、密碼與驗證碼，請聯繫專案開發者或系統管理員。

> ⚠️ 實際導入時請改為後端產生一次性驗證碼，並透過 email/SMS 發送給神秘客。

## 部署與客製化

1. 將 `index.html` 上傳至任意靜態網站服務或單機伺服器即可使用。頁面預設載入 Tailwind CDN，需保持可連線狀態。
2. 在 `index.html` 中調整 `MOCK_APPROVED_USERS`、`MOCK_SLOTS_DATA` 等模擬資料，即可快速修改可登入帳號與可預約時段。
3. 若要串接 n8n 或其他後端，可將登入、預約與註冊處的模擬 `setTimeout` 改為實際的 API 呼叫，並在成功回應後維持目前的 UI 更新流程。
4. 需要正式的雙重驗證時，請於後端產生驗證碼並透過 email/SMS 發送，再將回傳結果與目前的 6 位數欄位連動。

## 與 n8n 整合建議

- 於 n8n workflow 接收預約或註冊表單時，可新增帳號審核、時段鎖定、重複場次檢查等節點，確保單機部署也能避免重複填寫。
- 可使用 n8n 產生一次性驗證碼並寄送給神秘客，再於登入流程中驗證，取代目前的模擬碼。
- 若要導入原先的多階段回報表單，可在登入後導向另一個頁面或嵌入額外的表單卡片，以便重用現有的身分驗證流程。
